# 当前通信与触觉配置

## 通道分配

- 右侧触觉传感器：USART1，PA9 TX、PA10 RX，460800-8-N-1。
- 左侧触觉传感器：USART2，PD5 TX、PD6 RX，460800-8-N-1。
- 每片触觉传感器按两个单元轮询：
  - 上部地址：`0x6D`
  - 下部地址：`0x36`
- RS485 调试口：USART10，460800-8-N-1。
  - `RS485_TX1 = PE3 = USART10_TX = AF11`
  - `RS485_RX1 = PE2 = USART10_RX = AF4`
  - `RS485_EN = PC13`
- CAN 与 RS485 通过跳线帽复用外部端子。当前调试阶段默认关闭 CAN：`GRIPPER_CAN_ENABLE=0`。

## 触觉数据软件分层

触觉链路按三层设计：

```text
Platform/tactile_uart_transport
  只负责 UART DMA 写读、超时和底层错误统计。

Application/Tactile/tactile_protocol
  只负责 2 字节读取命令、帧头、地址、命令字、通道数、校验和 96 Byte 数据帧解析。

Application/Tactile/tactile_service + tactile_data
  负责双传感器轮询、快照发布、历史缓冲、帧率统计和调试输出。
```

后续自适应触觉力控只需要调用：

```c
TactileSnapshot_t snapshot;

if (TactileService_GetSnapshot(&snapshot))
{
  /* 根据 snapshot.right / snapshot.left 做接触、力控或滑移判断。 */
}
```

力控代码不要直接访问 USART1、USART2 或协议解析细节。

## 调试输出

RS485 调试口使用硬件 USART10。发送前 PC13 拉高，使能 485 驱动；发送完成并等待 UART TC 后，PC13 拉低释放总线。

PC 端串口参数：

```text
460800 baud, 8 data bits, no parity, 1 stop bit
```

触觉摘要输出：

```text
touch: sample,mask,ru_n,ru_t,rl_n,rl_t,lu_n,lu_t,ll_n,ll_t
```

触觉帧率输出：

```text
touchfps: ru,rl,lu,ll
```
